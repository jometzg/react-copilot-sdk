# React Copilot SDK Frontend - Project Plan

## Executive Summary

Build a web application that allows users to upload files (CSV), select custom AI agents, run them, view streamed responses in real-time, and download generated artifacts. The implementation prioritizes:

- **Security**: Credentials stay server-side, not exposed to frontend
- **Scalability**: Support multiple concurrent users
- **Extensibility**: Easy to add new agents without code changes
- **Real-time UX**: Stream responses incrementally as they arrive

## Core Requirements

### Requirement 1: File Upload
**User Story:** As a user, I want to upload a CSV file to process

**Acceptance Criteria:**
- ✅ Frontend provides file input (accept `.csv` only)
- ✅ Validation: Only CSV files accepted
- ✅ No size limit during MVP (20 MB soft limit)
- ✅ File stored temporarily during processing
- ✅ API returns 202 Accepted with session ID

**Implementation:**
- Frontend: `<input type="file" accept=".csv" />`
- Backend: Multer with memory storage
- Route: `POST /api/process`
- Response: `{ sessionId, status }`

---

### Requirement 2: Agent Selection
**User Story:** As a user, I want to select from available agents before processing

**Acceptance Criteria:**
- ✅ Frontend shows list of available agents
- ✅ Agents auto-discovered from `.github/agents/`
- ✅ Each agent has name, description, ID
- ✅ Can select exactly one agent
- ✅ Agent list loads on app startup
- ✅ First agent pre-selected by default

**Implementation:**
- Backend auto-discovers agents: `loadAgents()` scans `.github/agents/*.json`
- API: `GET /api/agents` returns `{ agents: AgentSummary[] }`
- Frontend: `<select>` dropdown populated from API
- Validation: Selected agentId must exist before processing

**Sample Agents:**
- `csv-analyst`: Analyzes CSV structure and content
- `data-quality-auditor`: Checks for data quality issues

---

### Requirement 3: Button Invocation
**User Story:** As a user, I want to click a button to start agent processing

**Acceptance Criteria:**
- ✅ Submit button triggers processing
- ✅ Button disabled if file not selected OR no agent selected
- ✅ Button disabled during processing
- ✅ "Submitting..." state shown while waiting
- ✅ User can see processing status
- ✅ Error messages if submission fails

**Implementation:**
- Frontend form: `<form onSubmit={onSubmit}>`
- Validation: `canSubmit = Boolean(file) && Boolean(selectedAgentId) && !submitting`
- Loading state: `setSubmitting(true/false)`
- Error handling: Display `runError` in UI

**Status Flow:**
- User clicks "Run Agent" → button disabled, "Submitting..." shown
- Backend returns 202 → status changes to "queued"
- Backend processes → status becomes "processing"
- Agent completes → status becomes "completed" or "error"

---

### Requirement 4: Response Display
**User Story:** As a user, I want to see agent responses stream in real-time

**Acceptance Criteria:**
- ✅ Responses appear incrementally (not all at once)
- ✅ Live updates without page refresh
- ✅ Response displayed in readable format
- ✅ Session ID visible for reference
- ✅ Processing status visible (queued/processing/completed/error)
- ✅ Error messages displayed if agent fails
- ✅ Stream disconnects gracefully

**Implementation:**
- Technology: Server-Sent Events (SSE)
- Frontend: `new EventSource('/api/process/{sessionId}/stream')`
- Backend: `res.setHeader('Content-Type', 'text/event-stream')`
- Event types:
  - `status`: Processing state change + message
  - `chunk`: Incremental response text
  - `artifact`: File generated (metadata only)
  - `completed`: Final session state
  - `agent-error`: Error message

**Streaming Payload:**
```
event: status
data: {"type":"status","status":"processing","message":"Processing started"}

event: chunk
data: {"type":"chunk","chunk":"First sentence. "}

event: chunk
data: {"type":"chunk","chunk":"Second sentence. "}

event: completed
data: {"type":"completed","session":{...}}
```

---

### Requirement 5: File Download
**User Story:** As a user, I want to download any files generated by the agent

**Acceptance Criteria:**
- ✅ Generated files listed with name and size
- ✅ Download link for each file
- ✅ Proper headers (Content-Disposition: attachment)
- ✅ Files accessible via URL
- ✅ No expiration during session lifetime
- ✅ Session-isolated storage (files per sessionId)

**Implementation:**
- API: `GET /api/process/{sessionId}/files/{fileId}/download`
- Response headers:
  ```
  Content-Disposition: attachment; filename="analysis.txt"
  Content-Type: text/plain
  Content-Length: [size]
  ```
- Storage: `data/outputs/{sessionId}/{fileId}`
- Tracking: `Map<sessionId:fileId, absolutePath>`

## Architecture

### Overview

```
┌─────────────────────────────────────────────────────────────┐
│                      React Frontend                         │
│  • User uploads CSV                                         │
│  • Selects agent from dropdown                              │
│  • Clicks "Run Agent"                                       │
│  • Sees response stream in real-time                        │
│  • Downloads generated files                                │
└─────────────────────────────────────────────────────────────┘
                            ↓ HTTP/SSE
┌─────────────────────────────────────────────────────────────┐
│                  Express Backend (Node.js)                  │
│                                                             │
│  POST   /api/process              → Multer upload handler   │
│  GET    /api/agents               → Agent discovery         │
│  GET    /:sessionId/stream        → SSE events              │
│  GET    /:sessionId/files         → List outputs            │
│  GET    /:sessionId/files/:id/... → File download           │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │           Core Services                             │   │
│  │ • agentLoader.ts    - Filesystem agent discovery   │   │
│  │ • copilotService.ts - Agent execution (mock/real)  │   │
│  │ • sessionStore.ts   - In-memory session state      │   │
│  │ • fileStore.ts      - CSV/output file management   │   │
│  │ • streamHub.ts      - SSE pub/sub                  │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### Technical Stack

**Frontend:**
- React 18.3.1 (component framework)
- TypeScript 5.6 (type safety)
- Vite 5.4 (build tool)
- CSS Grid (responsive layout)

**Backend:**
- Node.js 18+ (runtime)
- Express 5.2 (web framework)
- TypeScript 6.0 (type safety)
- Multer 2.1 (file upload)
- Zod 4.4 (schema validation)
- CORS 2.8 (cross-origin support)
- UUID/crypto (session IDs)

**Communication:**
- REST API for synchronous operations (POST, GET)
- Server-Sent Events (SSE) for streaming
- JSON payloads

### Security Decisions

1. **Server-Side Credentials**
   - Copilot SDK credentials never exposed to frontend
   - Backend uses Azure Identity/environment variables
   - Frontend has no auth concerns

2. **CORS Allowlist**
   - Only allow frontend origin(s)
   - Prevent unauthorized API access
   - Configured in `backend/src/server.ts`

3. **File Validation**
   - Only CSV files accepted (mime type check)
   - File size limits (20 MB default)
   - Isolated per-session storage

4. **Session Isolation**
   - Each upload gets unique sessionId (UUID)
   - Files stored per session
   - No cross-session data leakage

## Known Limitations

### By Design
- **Single file input** — Process one file at a time (batch processing future)
- **CSV only** — Other formats require adapter code
- **No file preview** — Users upload blindly
- **In-memory sessions** — Lost on backend restart
- **No multi-turn** — Single request → response cycle

### Compared to VS Code
- **No workspace context** — Agent sees only uploaded file
- **No skills auto-discovery** — Skills must be manually defined
- **No IDE integration** — No code editing or debugging
- **No extensions** — Cannot install additional tools
- **Stateless** — Each run independent, no conversation history
